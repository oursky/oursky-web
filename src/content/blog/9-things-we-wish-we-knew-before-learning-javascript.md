Is JavaScript going to conquer the world? This is a question we ask ourselves every time when there is a new and awesome JS framework coming out. May, one of Oursky’s growth hackers, asked our developers what they wish they knew before they learned JavaScript.

![JavaScript-was-not-Java](/images/blogs/6c40e5c663_67994e74974cebaa3ee2510b_chunyin_JSquote.png)

May’s Note: If there is a sentence that can set off all JavaScript and Java developers, it would be ‘Are Java and JavaScript related?’

![promise-can-act-like-functors](/images/blogs/77c6c3ab1c_67994e743f749d905256d6a2_boris_JSquote.png)

‍`Promise.resolve(1)   .then(n => n+10)   .then(n => n*n)   .then(n => console.log(n));   // 121`

![ECMAScript](/images/blogs/b423968e10_67994e74ff9dc23d3bfdaac4_louis_JSquote.png)

JavaScript is the most popular implementation of the [ECMAScript](http://www.ecma-international.org/publications/standards/Ecma-262.htm) standard.

The core features of JavaScript are based on the [ECMAScript](http://www.ecma-international.org/publications/standards/Ecma-262.htm) standard, but JavaScript also has other additional features that are not in the ECMA specifications/standard.

![JavaScript](/images/blogs/4edfd744a3_67994e74542c0b889ced6e08_Kenji_JSquote.png)

![JavaScript-is-conquering-the-world](/images/blogs/5b578c482c_67994e74f2137d56a8766068_may_JSquote.png)

According to [StackOverflow Developer Survey Result 2016](http://stackoverflow.com/research/developer-survey-2016), JavaScript is the most commonly used programming language on earth (85.3%).

![the-difference-between-the-equalities](/images/blogs/a20736b30d_67994e7496dda19f1d1da91a_david_JSquote.png)

This is an awesome table ( [https://dorey.github.io/JavaScript-Equality-Table/](https://dorey.github.io/JavaScript-Equality-Table/)) for understanding the difference between == & ===.  
May’s note: I wish I knew about this table when I first learnt JS. However, I have no ideas why the == table can be that strange.

![Firebug-is-over](/images/blogs/b0182db1d9_67994e749e5bbde3d2276b3d_brian_JSquote.png)

[Firebug](http://getfirebug.com/) is no longer maintained and there are other better debug tools (the built-in tool in chrome is good enough). When I started learning JS, [Firebug](http://getfirebug.com/) was a breakthrough, but debug tools are a norm now.
